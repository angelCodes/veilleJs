Installation Utilisation
VueJS est un framework JavaScript progressif utilisé pour développer des interfaces web interactives. Le focus est plus sur la partie vue, qui est le frontend. Il est très facile d'intégrer avec d'autres projets et bibliothèques.
  • Virtual DOM
    VueJS utilise le DOM virtuel, qui est également utilisé par d'autres frameworks tels que React, Ember, etc. Les modifications ne sont pas apportées au DOM, à la place une réplique du DOM est créée qui est présent sous la forme de structures de données JavaScript. Chaque fois que des changements doivent être faits, ils sont faits aux structures de données JavaScript et ce dernier est comparé avec le structure de données originale. Les modifications finales sont ensuite mises à jour vers le DOM réel, que l'utilisateur verra changer. C'est bon en terme d'optimisation, c'est moins cher et les changements peut être fait à un rythme plus rapide.
  • Data Binding
    La fonctionnalité de liaison de données aide à manipuler ou attribuer des valeurs aux attributs HTML, modifier la style, assigner des classes à l'aide de la directive de liaison appelée v-bind disponible avec VueJS.
  • Components
    Les composants sont l'une des caractéristiques importantes de VueJS qui permet de créer des éléments personnalisés, qui peut être réutilisé en HTML.
  • Event Handling
    v-on est l'attribut ajouté aux éléments DOM pour écouter les événements dans VueJS.
  • Computed Properties
    C'est l'une des caractéristiques importantes de VueJS. Cela aide à écouter les changements apportés au Éléments d'interface utilisateur et effectue les calculs nécessaires. Il n'y a pas besoin de codage supplémentaire pour ça.
  • Templates
    VueJS fournit des modèles HTML qui lient le DOM aux données de l'instance de Vue. Vue compile les modèles dans les fonctions virtuelles de rendu DOM. Nous pouvons utiliser le modèle des fonctions de rendu et pour ce faire, nous devons remplacer le modèle avec la fonction de rendu..
  • Méthode 1 || Utiliser directment balise script dans HTML.
    Allez sur le site d'accueil https://vuejs.org/v2/guide/installation.html de VueJS et Téléchargez la vue.js selon vos besoins et inclure dans HTML. Il existe deux versions à utiliser - la version de production et version de développement. La version de développement n'est pas minimisée, alors que la production oui.
  • Méthode 2 || EN Utilisant le CDN
    Nous pouvons également commencer à utiliser le fichier VueJS de la bibliothèque CDN. Le lien https://unpkg.com/vue donne la dernière version de VueJS. VueJS est également disponible sur jsDelivr (https://cdn.jsdelivr.net/npm/vue/dist/vue.js) et cdnjs (https://cdnjs.cloudflare.com/ajax/libs/vue/2.4.0/vue.js). Nous pouvons héberger les fichiers de notre côté, si nécessaire et commencer avec le développement de VueJS.
  • Méthode 3 || En utilisant npm.
    Pour les applications à grande échelle avec VueJS, il est recommandé d'installer en utilisant le paquet npm. Il est livré avec Browserify et Webpack avec d'autres outils nécessaires, qui aident à la développement. Voici la commande à installer en utilisant npm: npm install vue

  • Composants
    Vuejs 2 fournit des méthodes qui sont déclenchées à divers points depuis la création d'un composant jusqu'à la destruction du composant. C'est ce qu'on appelle le cycle de vie de l' instance , également connu sous le nom de Lifecyle du composant . Chaque instance de Vue passe par une série d'étapes d'initialisation lors de sa création - par exemple, elle doit configurer l'observation des données, compiler le modèle, monter l'instance sur le DOM et mettre à jour le DOM lorsque les données changent. Vous pouvez donc exécuter une logique personnalisée dans ces hooks. Ces crochets du cycle de vie sont : beforeCreate, created, beforeMount, mounted, beforeUpdate, updated, activated, deactivated, beforeDestroyet destroyed.
